# 客户端安全加固记录

## 2026-07-12 已完成

- Electron 固定为 `43.1.0`，生产依赖只保留两个固定版本的地址校验库；`npm audit` 结果为 0 个已知漏洞。
- 主窗口启用 sandbox、contextIsolation、CSP、导航限制、WebView 域名限制、权限拒绝、下载扩展名限制和 IPC 发送者校验。
- 打包后强制关闭 RunAsNode、NODE_OPTIONS、Node 调试等 Electron Fuses，并启用 ASAR 完整性与 OnlyLoadAppFromAsar。
- 客户端不再向渲染页返回服务密钥；正式包不携带试用密钥文件、授权端私钥或授权端源码入口。
- 已删除会从旧配置提取明文密钥的过时 `prepare-trial-config.cjs` 工具，并加入防回归门禁。
- 授权升级为 `MYM2 + MYLIC2`：设备 X25519 密钥由 DPAPI 保护，授权载荷使用 X25519/HKDF-SHA256/AES-256-GCM 加密，并由套装 Ed25519 私钥签名。
- 翻译磁盘缓存升级为 `TranslationCache/v3`：每条记录独立 AES-256-GCM 加密，密钥由 DPAPI 保护并绑定套装和数据根；旧明文缓存定向删除，不迁移。
- 主渲染页和指纹浏览器不再把译文明文写入 `localStorage`；页面侧只保留最多 1500 条或 15 MB 的进程内热缓存，重启后从机器绑定的加密 `TranslationCache/v3` 批量恢复。
- 主渲染页旧明文键在启动时立即删除；WhatsApp/Telegram 在 WebView 就绪时删除；Signal 在每个实例下次合法启动时删除。清理不迁移译文、不清除平台登录态。
- 自动企业聊天资产写入已移除，不会因归档重复调用翻译接口。
- Signal 使用随机本地端口、32 字节挑战、HMAC-SHA256、双向序列号、最大 1 MiB 帧和最多 64 个控制套接字；启动凭证有效 10 秒且只能使用一次。
- 普通翻译发送使用一次性翻译凭证，并在发送后核对新产生的我方气泡。
- 纯数字收款账号和 USDT 地址使用两次回车事务：第一次可信确认，第二次精确发送；凭证有效 120 秒且只能使用一次。
- 连续 5 次授权码错误后的永久锁死、全量数据清除与卸载逻辑保持启用，没有降级或开发环境豁免。

- 编号 12：主安装包与授权程序的 Authenticode 代码签名不在本轮处理。
- 本轮没有生成安装包，没有消耗套装 ID；源码随本次重构前本地 Git 恢复点归档，没有推送 GitHub。

## 无服务器边界

- 本地加固可阻止普通复制、远程网页脚本和未授权 Signal 启动，但无法绝对抵御已经取得 Windows 管理员或当前用户进程注入能力的恶意程序。
- 未做代码签名时，攻击者若能同时替换整个可执行文件和应用包，仍可能绕过仅针对原程序的完整性检查；这是本轮明确保留的风险。
- Signal 动态脚本入口目前只允许受 ASAR 完整性保护的本地主渲染页调用，远程平台页面无法调用。迁移为完全类型化命令仍是后续纵深加固项。
