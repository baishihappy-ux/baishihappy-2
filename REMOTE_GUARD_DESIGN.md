# 御前侍卫：Windows 远控守卫设计

更新时间：2026-07-17

## 目标

御前侍卫用于补充 BitLocker 和客户端锁屏的在线防护空白：电脑已经开机、Windows 用户已经登录时，发现高置信远控迹象后先遮蔽聊天内容，再触发全屏声光警报和 Windows 锁屏。

它不是杀毒软件，也不承诺识别所有未知木马、内核驱动、进程注入或自制录屏程序。WebSocket、HTTPS、QUIC 等只是通信方式，不能单独证明远控行为。

## 巡检策略

- 快速巡检周期：8 秒。
- 快速巡检内容：Windows 会话状态，以及 TCP 活动连接中的本机 VNC 服务端口。
- 快速命令超时：900 毫秒。
- 远控进程补充巡检周期：5 分钟。
- 进程清单命令超时：1.2 秒；失败后不连续重试，界面显示“巡检受限”。
- 不使用高频 PowerShell、WMI、全盘扫描、抓包或持续录屏。
- 巡检状态只保存在内存，不记录聊天、用户名、密钥、网络地址和消息内容。

## 风险分级

### 已确认远控

- 当前或其他活动 Windows 会话为 RDP 远程桌面会话。
- 本机 VNC 服务端口 5800 至 5999 存在已建立连接。

已确认远控会触发完整安全响应。

### 可疑程序

检测到 AnyDesk、TeamViewer、RustDesk、ToDesk、向日葵、Windows 快速助手、Windows 远程协助、Chrome 远程桌面、ScreenConnect、Splashtop、Supremo、DWService、MeshCentral、Parsec 或常见 VNC 服务进程时，显示黄色警示。

仅有进程存在不能证明远控已经建立，因此不会直接锁定 Windows，避免将合法安装但未连接的软件误判成入侵。

## 安全响应时序

1. 第 0 秒：主进程锁定工作区可见性门禁。
2. 第 0 秒：隐藏 Signal 窗口；Signal 未确认隐藏时沿用失败关闭策略。
3. 第 0 秒：主窗口内的 WhatsApp、Telegram 和 Signal 内容被全屏遮罩。
4. 第 0 秒：每个显示器创建独立的置顶安全警报窗口，并循环发出系统警报声。
5. 第 3 秒：主进程通过 Windows 系统目录中的 `rundll32.exe` 调用 `LockWorkStation`。
6. Windows 解锁或系统唤醒后立即复查；远控仍存在时重新进入 3 秒锁屏响应。
7. 远控迹象消失后仍保持事件锁定：已设置客户端锁屏密码的用户必须验证密码；未设置密码时必须在警报界面确认恢复。

远控存在时，锁屏密码不会释放工作区。普通关闭按钮也不能关闭处于警报状态的主窗口。

## 安全边界

- 同一 Windows 用户权限下的恶意程序可以伪装名称、结束进程或避开已知规则。
- 管理员权限或内核权限攻击者可以绕过用户态守卫。
- 第三方远控软件常使用长期中继连接，仅根据外连无法可靠判断是否正在观看屏幕。
- 系统静音、音频设备切换或远程音频策略可能使警报声不可听，因此 Windows 锁屏才是主要阻断措施。
- 必须继续配合 Windows Defender、防火墙、BitLocker、普通用户权限和客户端锁屏使用。

## 代码与打包边界

- 策略解析：`electron/remote-guard-core.ts`
- Windows 巡检与锁屏请求：`electron/remote-guard.ts`
- 主进程状态机、警报窗口和工作区门禁：`electron/main.ts`
- 受限 IPC：`electron/preload.ts`、`electron/preload.cjs`、`electron/shared.ts`
- 左侧入口、状态面板和警报界面：`src/App.vue`、`src/styles.css`
- 自动回归：`tools/test-remote-guard.mjs`
- 客户包必须显式包含 `dist-electron/remote-guard.js` 和 `dist-electron/remote-guard-core.js`。

## 已完成的自动验证

- 活动 RDP 会话触发红色警报。
- 已断开的 RDP 会话不触发锁机。
- 仅运行已知远控程序只显示黄色警示。
- 本机 VNC 活动连接触发红色警报。
- 本机作为 VNC 查看端连接其他电脑时不误判为被控端。
- 每 8 秒快速巡检、每 5 分钟进程补查和 3 秒 Windows 锁屏参数受测试约束。
- 新模块必须进入普通客户包和套装客户包。
- 完整项目安全回归和 Electron 窗口烟雾测试通过。

## 仍需人工验收

- 在受控测试电脑上建立真实 RDP 会话，确认第 0 秒遮罩、第 3 秒 Windows 锁屏和重新登录后的事件锁定。
- 在安装 VNC 服务端的测试电脑上建立真实入站连接，确认同样的响应。
- 分别验证黑金和粉色主题下的状态面板、全屏警报和声音。
- 验证多显示器上的每屏警报，以及系统静音和不同音频设备条件下的表现。
