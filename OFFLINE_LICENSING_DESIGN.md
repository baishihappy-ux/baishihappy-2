# 离线授权体系设计

## 产品边界

- 客户端无maoyi服务器依赖，不需要在线授权服务器。
- 客户端直接调用配置的翻译服务。
- 客户端单独销售时，供应商保留授权程序并代为签发授权。
- 全套销售只交付客户端安装包、授权程序和加密签发密钥，不交付源码。
- 全套产品以压缩包交付，授权端和客户端是一一配套的离线体系。

## 每套独立密钥

每套全套产品必须生成独立的 Ed25519 签名密钥对：

- 授权程序持有加密后的签发私钥。
- 客户端内置对应验证公钥。
- 授权程序使用私钥签发 `license.dat`。
- 客户端只负责验签，不能自行签发授权。
- 不同套装的授权文件不能混用。

授权载荷必须包含 `issuerId`、`productId` 和 `keyId`。客户端同时验证签名及这三个标识。

## 套装 ID 池和密钥生成

开发者环境维护 10000 条不重复的 9 位数字套装 ID。套装 ID 池保存在 `.package-secrets` 下，不进入 Git。

每次开发者执行整套打包流程时，必须消耗 1 个未使用的套装 ID。无论后续打包成功、失败、中断或作废，该套装 ID 都不得回收、不得重新使用。

套装 ID 的作用不是直接充当密钥，而是参与并锁定本次套装密钥生成流程：

- 从 ID 池取出 1 个未使用套装 ID。
- 立即把该套装 ID 标记为 `consumed`（已消耗）。
- 使用这个唯一套装 ID 为当前要打包的套装生成一套独立签名密钥对。
- 生成的 `suiteId`、`keyId`、公钥指纹、加密私钥指纹、创建时间和输出目录必须永久绑定记录。
- 如果打包失败，该套装 ID 和对应生成记录保持为失败或作废状态，不允许回到可用池。

强制校验：

- ID 池内 10000 条套装 ID 必须全部唯一。
- 已消耗套装 ID 不得再次用于生成密钥。
- 不同套装 ID 不得出现相同公钥指纹。
- 不同套装 ID 不得出现相同加密私钥指纹。
- 输出目录已存在时禁止覆盖。

## 设备绑定

1. 客户端首次运行时生成设备加密密钥对。
2. 设备私钥使用 Windows DPAPI 保护。
3. 本机码包含硬件摘要、设备公钥和产品标识。
4. 授权程序使用设备公钥加密授权载荷。
5. 客户端使用设备私钥解密，并验证硬件摘要。
6. `license.dat` 复制到其他电脑后不能使用。

## 授权端输入与输出

授权端输入：

- 本机码
- 用户名
- 密钥，界面文案固定为“请输入密钥”，不显示具体模型或服务商名称
- 授权天数

授权端输出：

- `license.dat`
- 授权码
- 用户名
- 生效时间
- 到期时间
- License ID

授权码就是 `license.dat` 文件内容的文本形态，用于客户复制输入。授权程序界面必须提供“复制授权码”按钮。

客户端输入授权码后，在当前程序数据目录自动生成 `license.dat`。授权码和 `license.dat` 是同一份加密授权载荷的两种使用方式。

授权程序和客户端都不生成客户登录密码。客户端只设置一次用户名，并通过授权码进入。

客户端本地登录规则：

- 用户名只支持设置一次，设置后无法修改。
- 设置用户名时显示提示：“用户名仅支持设置一次，设置后无法更改。”
- 未设置用户名时点击复制机器码信息，提示：“请先设置用户名。”
- 设置用户名后，点击复制会复制本机码和用户名。
- 不设置客户登录密码，不保存客户登录密码哈希。
- 有效授权会在后续启动时自动验证并进入。

## 授权载荷

载荷至少包含：

- 授权格式版本
- License ID
- `issuerId`、`productId`、`keyId`
- 本机硬件摘要和设备公钥摘要
- 用户名
- 加密服务密钥
- 签发、生效和到期时间
- 授权天数
- 功能列表

当前授权文本前缀为 `MYLIC2`。授权端为每次授权生成临时 X25519 密钥，结合客户端本机码内的设备 X25519 公钥，经 HKDF-SHA256 派生 AES-256-GCM 密钥；完整加密信封再由套装 Ed25519 私钥签名。

授权码文本不得是可读 JSON 明文。客户用文本编辑器打开授权码或 `license.dat` 时，只能看到加密后的授权文本，不能直接读出密钥、机器码、用户名、有效期或功能列表。

## 授权程序启动保护

- 授权程序首次打开时必须先设置授权程序启动密码。
- 首次设置密码时显示提示：“该密码用于保护授权程序，请务必牢记。密码遗失后无法进入授权程序。”
- 启动密码使用 `scrypt` 加盐派生密钥，不保存明文；固定参数为 `N=131072`、`r=8`、`p=1`、32 字节随机盐。
- 签发私钥和套装配置使用 AES-256-GCM 加密，外层再由 Windows DPAPI 保护。
- 密码正确后才允许解密私钥并进入授权界面。
- 程序退出时清除内存中的私钥和派生密钥。

密码连续错误锁定阶梯：

| 锁定轮次 | 连续输错次数 | 锁定时间 | 提示 |
|---|---:|---:|---|
| 第 1 轮 | 3 次 | 10 分钟 | 显示倒计时 |
| 第 2 轮 | 再错 3 次 | 1 小时 | 倒计时并显示“请联系供应商” |
| 第 3 轮 | 再错 3 次 | 6 小时 | 倒计时并显示“请联系供应商” |
| 第 4 轮 | 再错 3 次 | 24 小时 | 倒计时并显示“请联系供应商” |
| 第 5 轮及以后 | 每错 3 次 | 48 小时 | 倒计时并显示“请联系供应商” |

- 锁定状态使用 DPAPI 保护并持久保存。
- 重启程序或电脑不能清除锁定状态。
- 锁定期间禁用密码框和登录按钮。
- 输入错误时显示剩余尝试次数。
- 检测系统时间回拨；时间异常时继续锁定。
- 成功登录后错误次数和锁定等级归零。

## 离线授权边界

- 全套购买方拥有其套装的离线签发权。
- 无服务器模式不能可靠限制全套购买方签发数量。
- 私钥丢失后无法继续为原套装签发；私钥泄露后必须更换客户端公钥和套装版本。
- 服务密钥在无服务器客户端运行时无法做到绝对不可提取，但必须使用 DPAPI 保存、禁止返回页面、禁止写日志。

## 已交付套装升级

- 已交付客户的版本升级不是新套装生成，不消耗新的套装 ID。
- `003` 客户升级必须复用套装 `183105912` 的原始 `suiteId`、`keyId` 和完整密钥对。
- 严禁仅使用相同数字 ID 重新生成密钥；新密钥无法验证原有 `license.dat`。
- 升级只替换程序文件，保留数据位置指针、授权、设备身份、多开配置和三个平台登录态。
- 第一个加密缓存版本不迁移旧明文译文，只定向删除 `TranslationCache` 和 `df.translation.renderCache.*`，然后写入一次性清理标记。
- 新客户交付仍按新套装流程永久消耗一个新 ID并生成独立密钥对。
- `003` 从旧 `DFLIC1` 升级到 `MYLIC2` 时必须使用原套装密钥重新签发一次授权码；套装 ID 和密钥不变，但旧格式授权文本不继续兼容。
